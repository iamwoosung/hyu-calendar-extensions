# Port Specification

## Overview

| Port  | Protocol   | Service               | Component         | Description                              |
|-------|------------|-----------------------|-------------------|------------------------------------------|
| 3000  | HTTP       | Server (Node.js)      | server/           | REST API (로그인, 동기화 요청, 세션 관리)     |
| 4000  | HTTP / WS  | Supabase Realtime     | docker (realtime) | WebSocket 실시간 이벤트 push (익스텐션 구독)  |
| 5432  | TCP        | PostgreSQL            | docker (postgres) | 메인 데이터베이스                            |
| 5672  | AMQP       | RabbitMQ              | docker (rabbitmq) | 메시지 큐 브로커 (Server → Consumer 작업 전달) |
| 15672 | HTTP       | RabbitMQ Management   | docker (rabbitmq) | RabbitMQ 관리 UI (http://{HOST}:15672)  |

## Details

### 3000 — Server API
- 담당: `server/`
- 접근 주체: Chrome Extension

### 4000 — Supabase Realtime
- 담당: realtime 서비스
- 프로토콜: HTTP Upgrade → WebSocket (`ws://{HOST}:4000/socket`)
- 접근 주체: Chrome Extension
- 용도: Consumer가 DB SyncStatus 변경 시 익스텐션에 즉시 push

### 5432 — PostgreSQL
- 담당: postgres 서비스
- 접근 주체: Server, Consumer, Supabase Realtime (Docker 내부 네트워크)

### 5672 — RabbitMQ AMQP
- 담당: RabbitMQ 서비스
- 접근 주체: Server, Consumer
- 프로토콜: AMQP 0-9-1

### 15672 — RabbitMQ Management UI
- 담당: RabbitMQ 서비스
- 접근 주체: 개발자
- 용도: 큐 상태, 메시지 수, 처리량 모니터링
